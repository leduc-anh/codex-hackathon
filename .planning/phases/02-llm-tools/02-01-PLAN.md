## Tasks

- [ ] Inspect existing tool/action files and preserve any user changes already in progress.
- [ ] Implement or finish `src/lib/tools/score_fit.ts` as pure deterministic TypeScript.
- [ ] Ensure fit output is qualitative only and never includes or renders a percentage.
- [ ] Add or align typed action result helpers so validation/search/LLM failures return degraded typed results.
- [ ] Implement or stub `log_feedback` only to the MVP depth needed by the contract; do not build a mentor UI.
- [ ] Add focused tests for `score_fit` and any shared result/error helper touched.
- [ ] Update `docs/MVP_TASK.md` only for tasks whose acceptance criteria are actually met.

## Must Haves

truths:
- `score_fit` computes fit without calling the LLM.
- Criteria and gaps preserve source grounding expectations.
- Tool errors do not crash the caller path.
- No new dependency is added.

prohibitions:
- No percentage or success-probability number is introduced.
- No raw unvalidated JSON enters logic.
- No vector memory, dashboard, auth, or B2B UI work is added.
